My experience doing Code review (WIP)


In this part of my blog, I'll share what I have learned in CTF's about doing Code Review to Web Applications. I'm doing this for three main reasons:

  • Besides being a great skill to have for a job or for Playing CTFs, I think it is fun doing code reviews of web apps
  • We have a lot of guides on how to do code reviews but there are things that you only learn by experience.
  • I want to share and have a journal of what I'm learning.

I hope someone finds it useful.


Before starting

Before starting I just want to state some things:

  • I will not be using any kind of automated tools to look for vulnerabilities in code, let's just omit that part for now.
  • I will be sharing my experience or examples from CTF challenges not from real projects.
  • I will try to not repeat things from other guides or common vulnerabilities that are pretty obvious.
  • I want to share how I'm developing this skill and my methodology, I'm not reinventing the wheel so I want to share also those parts of how I do things.
  • I'm going to separate the code review work into sections that you can find in this blog.
  • I'm not focusing on a language, don't worry too much about it! We can do it!

Now let's start with this journey!
